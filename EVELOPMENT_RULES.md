# 🛠️ MoEatzy AI 개발 지침 및 코딩 규칙 (Development Rules)

본 문서는 AI 코딩 에이전트가 코드를 제멋대로 파편화하거나 하드코딩하지 않도록 강제하는 엄격한 약속입니다. 모든 코드 생성 시 이 가이드라인을 최우선 순위로 준수하십시오.

## 1. 아키텍처 및 소스코드 분리 규칙
- **하드코딩 절대 금지**: `index.html`에 화면 내용 전체를 길게 작성하지 마십시오.
- **역할의 철저한 위임**:
  - `index.html`: 레이아웃 뼈대와 탭 바 구조만 정의합니다.
  - `style.css`: 모든 스타일, 글래스모피즘 효과, 애니메이션을 전담합니다.
  - `data.js`: 시연에 필요한 모든 식품 목록, 레시피, 리포트 수치 데이터를 객체/배열 형태로 독점 관리합니다.
  - `app.js`: 탭 전환 시 화면을 동적으로 그리는 핸들러(DOM Rendering)와 유저 클릭 이벤트 로직만 작성합니다.

## 2. 상태 관리 및 인터랙션 약속
- **화면 전환**: 하단 탭 클릭 시 `app.js`에서 해당 화면에 맞는 DOM 요소를 생성하여 `#content` 영역에 주입하는 Single Page Application(SPA) 방식을 취하십시오.
- **시연용 타이밍**: 스캔 및 업로드 액션 시, 유저에게 테크니컬한 재미를 주기 위해 반드시 `setTimeout`을 활용한 **2초간의 고의적 로딩 상태**를 구현하고 로딩 바를 화면에 노출하십시오.
- **성공 보상 시각화**: '레시피 완료' 버튼을 누르면 화면 중앙에 이펙트 애니메이션(CSS 혹은 가벼운 JS 활용)이 최소 1.5초간 노출된 후 인벤토리에서 해당 재료가 소모(삭제)되도록 구현하십시오.

## 3. 가짜 결제(Fake Door) 실험 구현 규칙
- 스마트 바잉 화면의 유료 구독 버튼 결제 이벤트 발생 시, 실제 결제 창이나 외부 페이지 링크를 연결하지 마십시오.
- `app.js` 내부에 모달 제어 함수(`openModal()`, `closeModal()`)를 작성하여 깔끔하고 세련된 다크 투명 배경의 안내 팝업창을 중앙에 띄우십시오. 팝업 내부에는 닫기(`X`) 버튼이 명확히 작동해야 합니다.
